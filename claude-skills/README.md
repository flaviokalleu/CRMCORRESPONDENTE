# Claude Code Skills System

A structured set of behavioral rules and coding guidelines organized by domain.

## Structure

| Folder | Purpose |
|---|---|
| `core/` | Token optimization, clean architecture, code quality |
| `security/` | OWASP, auth, input validation, secrets management |
| `backend/` | API design, database, services, error handling |
| `frontend/` | React patterns, UI/UX, state management, performance |
| `automation/` | Workflow automation, scripts, CI triggers |
| `testing/` | Unit, integration, E2E, mocking strategies |
| `devops/` | Docker, CI/CD, monitoring, infrastructure |
| `ai/` | AI integration, prompt engineering, embeddings |
| `advanced/` | Debugging, profiling, refactoring, migrations |

## Usage

Each `.md` file contains behavioral rules that guide Claude Code when working on tasks in that domain. Reference specific skills by path when needed.
